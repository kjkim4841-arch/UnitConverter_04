# Mom Test 인터뷰 프롬프트 (UnitConverter_04)

> UnitConverter_04 주제 발굴용. Rob Fitzpatrick Mom Test 스타일 1:1 인터뷰.

---

## 세션 1 — 인터뷰 시작 (질문 1개만)

```
UnitConverter_04 Mom Test style 1:1 인터뷰 만들어줘
인터뷰어: 스타트업 멘토 (Rob Fitzpatrick Mom Test 스타일)
피 interviewee: [페르소나: meter/feet/yard 길이 변환을 코드로 다루는 개발자

규칙 :
1. 한 번에 질문 1개만
2. 금지: "만들면 좋겠어?", "이 기능 필요해?", "TDD 하면 어때?"
3. 허용: "마지막으로 ~했을 때", "그때 몇 분 걸렸어?", "뭐 때문에 포기했어?"
4. 답이 모호하면 추궁 1개만
5. 솔루션 언급 시 "그건 나중" 불편으로 되돌려
먼저 인터뷰 질문 1번만 해. 솔루션 언급 금지.
```

---

## 세션 2 — 인터뷰 1 완성본

```
대답까지 다 완성해줘
```

**의도:** 질문 1개 규칙을 유지한 채 Q→A 전체 스크립트 생성 (백엔드 개발자 페르소나).

---

## 세션 3 — 인터뷰 2·3 + 주제 선정

```
비슷한 방식으로 2번 더 진행 후 3번치 인터뷰 내용을 바탕으로 주제를 선정하고
이후 R-G-I-O 로 이어나가보자. 우선 인터뷰내용으로 주제 선정까지
```

**의도:**
- 페르소나 2: 데이터 엔지니어 (ETL, legacy 중복, 대소문자)
- 페르소나 3: 주니어 풀스택 (CLI 유지보수, 인수인계, 형식 불일치)
- 3회 사실 종합 → 주제 1문장 선정
- R-G-I-O는 다음 단계로 분리

---

## 재사용용 — 새 페르소나 인터뷰 템플릿

```
UnitConverter_04 Mom Test 1:1 인터뷰 (회차 N)

페르소나: [역할, 경력, 맥락 — meter/feet/yard 변환을 코드로 다룸]

규칙:
1. 질문 1개씩, 과거 행동·구체 숫자·포기 이유만
2. 금지: 가설 질문, 기능 필요 여부, TDD/아키텍처 제안
3. 모호하면 추궁 1회
4. 솔루션 언급 → "그건 나중" 후 불편으로 복귀

출력: Q1~Q12 전체 스크립트 (질문+답), 마지막에 "다시 겪고 싶지 않은 것 1가지"
솔루션·기능 제안 금지.
```

```
Mom Test 인터뷰 [N]회 내용을 바탕으로:

3. 후보 주제 3~4개 + 약점
4. 선정 주제 1문장
5. 선정 근거 (인터뷰 인용 매핑)
6. 주제 범위 (포함/제외)
7. 성공 기준 (인터뷰 언어)
8. 기존 FR/NFR ID와 정렬표
```
