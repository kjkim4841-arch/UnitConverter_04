## 2. 선정 주제 (1문장)

> **meter / feet / yard 길이 변환을, 비율과 검증 규칙이 한곳에 고정된 테스트 가능한 모듈로 만든다.**

---

## 3. 선정 근거 (인터뷰 → 주제 매핑)

| 인터뷰 사실 | 주제가 겨냥하는 것 |
|-------------|-------------------|
| 동일 변환이 여러 파일·분기에 **다른 비율** (1·2) | 비율 **한곳**, meter 기준 |
| feet↔yard 직접 계산 시 0.01~0.02 어긋남 (1·3) | **meter 경유** 규칙 고정 |
| `meter:2.5`/`meter:1`만 손검증, yard·음수는 사후 터짐 (1·3) | **테스트 가능** = 경계까지 계약화 |
| yard/cubit/inch 추가마다 2시간~1일 + 회귀 불안 (1·3) | 모듈 단위 **회귀 확인** |
| 금요일 2.5h, 새벽 온콜 55분, 월합계 3h 추적 (1·2) | "작은 수정" 비용 **사전 차단** |

## 4. 주제 범위

### 포함 (P0)

- 입력: `unit:value` (예: `meter:2.5`)
- 단위: meter, feet, yard
- 변환: 1m = 3.28084ft, 1m = 1.09361yd; **feet↔yard는 meter 경유**
- 검증: 음수 거부, 잘못된 형식, 미지 단위
- 출력 정밀도: 소수 4자리 (예: 8.2021, 2.7340)
- 구조: 비율·변환·검증이 **한 모듈(계층)에서 추적 가능**

### 제외 (이번 주제 밖 → P1·별도)

| 항목 | 제외 이유 |
|------|-----------|
| JSON/YAML 설정 (EXT-01) | 인터뷰에 과거 필요 행동 없음 |
| 동적 cubit 등록 (EXT-02) | 인터뷰 3에서 **불안**만 확인, 설정 외부화 요구 없음 |
| json/csv/table 출력 (EXT-03) | 인터뷰 3은 **4자리 table** 불일치만 언급 |
| ECB·TDD·OCP **용어** | 주제 문장에 넣지 않음 — 구현 단계에서 품질 요구로 연결 |

## 5. 주제 성공 기준 (인터뷰 언어)

다음이 **재발하지 않으면** 주제 달성:

1. yard 한 줄 추가가 금요일 오후 **2시간+**로 커지지 않는다.
2. feet→yard가 QA/동료 **0.01 차이 제보 전에** meter 경유로 검증된다.
3. `meter:-1`, `meter`(콜론 없음) 등이 CS/온콜 **이후가 아니라** 모듈 계약에서 걸러진다.
4. cubit/inch 추가 시 "동작하면 PR"이 아니라 **meter/feet/yard 회귀가 자동 확인**된다.
